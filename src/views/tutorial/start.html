Getting started

Beril engine use Component Entity System - a pattern widely used in game development.

Few common things to know before you can start work with Beril:

Application Mode

this is where you define what systems should be included in game, their settings and so on. You define your application mode class inherited from Beril.ApplicationMode and then pass it to your application either in init function:

application.init({mode: MyApplicationMode})

or using setMode property of application:

application.setMode(MyApplicationMode)

Default Pawn
The default pawn class to be used. Think about this like a main character class or something with appached camera to it. Without any active camera you would not see any thing. Beril.DefaulPawn alriedy have a camera attached to it.

There are few simple steps to make you first Beril application:

  1. Create new application instance - application = new Beril.Application();
  2. Init application instance with settings
  3. run your application - application.run()

This is a basic Beril set up that will create empty application with default application mode and default PlayerCharacter which is nothing but an empty mesh with attached camera to it.

Now it's time to add few objects. Say we want to have a rotating cube. For this we will create a new class inherited from Beril.GameObject. The constructor for this class accept array of components constructors to be used in this object. If we want this object to be visible - we need render system to process it and thus we need to add render component to this object. Also we want our cube to have some rotation. There is a translation system that apply translation to position, rotation and scale components.

class extends Beril.GameObject{
  constructor(){
    super([
      Beril.RotationComponent,
      Beril.PositionComponent,
      Beril.ScaleComponent,
      Beril.RenderComponent,
      Beril.TranslationComponent,
    ]);
    this.name = "Cube";
  }
}
